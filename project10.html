Data breach alert system

Objective or Description
Project Description: The Data Breach Alert System is a comprehensive tool designed to monitor various public databases and forums for data breaches and alert users if their information appears in a new breach. The system will use web scraping to gather data, machine learning algorithms for data analysis, and a user-friendly interface for users to interact with the system.
Tools, Technologies, concepts and workflow
Web Scraping: Use web scraping tools to gather data from various public databases and forums where data breaches are often shared. You could use Python libraries like BeautifulSoup or Scrapy for this.
Data Processing: Clean and organize the scraped data. This could involve removing irrelevant information, sorting the data, and more. Python’s Pandas library can be useful for this.
Data Analysis: Analyze the processed data to identify if any user information has been compromised. This could involve pattern recognition or even machine learning algorithms. Python’s Scikit-learn library can be useful for this.
User Interface: Develop a user interface where users can enter their information (like email addresses) to be monitored. This could be a web application built with HTML, CSS, and JavaScript.
Alert System: If the system identifies any compromised user information, it should alert the user. This could be an email alert or a notification in the user interface.
Database Management: Store the user information and any identified breaches securely. You could use a database system like MySQL or MongoDB for this.
APIs: Consider using APIs that provide access to known data breaches like “Have I Been Pwned API”.
Security: Ensure that all user data is handled and stored securely. This is crucial when dealing with sensitive information.
Role of team members
Student 1 - Web Scraping and Data Collection: This student will be responsible for developing and implementing the web scraping tools that will gather data from various public databases and forums. They will use Python libraries like BeautifulSoup or Scrapy.
Student 2 - Data Processing and Analysis: This student will clean and organize the scraped data. They will also analyze the processed data to identify if any user information has been compromised. They will use Python’s Pandas library for data processing and Scikit-learn library for data analysis.
Student 3 - User Interface and Alert System: This student will develop a user interface where users can enter their information to be monitored. They will also implement the alert system that will notify users if their information is found in a new data breach. They will use HTML, CSS, and JavaScript for the user interface.
Student 4 - Database Management and Security: This student will be responsible for storing the user information and any identified breaches securely. They will also ensure that all user data is handled and stored securely. They could use a database system like MySQL or MongoDB for this.
Use Cases
User Registration: Users can register to the system by providing their email address and other optional information.
Data Breach Monitoring: Once registered, the system will continuously monitor various public databases and forums for any signs of the user’s information.
Data Breach Alert: If the system identifies any compromised user information, it will alert the user via an email alert or a notification in the user interface.
User Dashboard: Users can log in to their dashboard to see the status of their data breach monitoring and any alerts.
Data Update: Users can update their information in the system to be monitored.
