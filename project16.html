City Guide

Objective or Description
The project aims to develop an interactive city guide that provides users with information about various cities. The guide will include details about tourist attractions, restaurants, hotels, and other points of interest. Users can search for a city and get a comprehensive guide about what the city has to offer.
Tools, Technologies and concepts
APIs (Google Places API or OpenStreetMap API): To collect data about various cities.
Backend Framework (Node.js or Django): To develop the backend of the application.
Frontend Technologies (HTML, CSS, JavaScript): To develop the user interface of the application.
Frontend Framework (React or Angular): To develop a dynamic and interactive user interface.
Testing Framework (Jest or Mocha): To test the application.
Git: For version control and collaboration.
Role of team members
Student 1 - Data Collection and Processing: This student will be responsible for collecting and processing data about various cities. They can use APIs like Google Places API or OpenStreetMap API for this.
Student 2 - Backend Development: This student will develop the backend of the application, which will handle requests and responses, manage data, and integrate with the APIs. They can use a backend framework like Node.js or Django for this.
Student 3 - Frontend Development: This student will develop the user interface of the application. They will use HTML, CSS, and JavaScript for this. They can also use a frontend framework like React or Angular for this.
Student 4 - Testing and Optimization: This student will be responsible for testing the application to ensure it works correctly. They will also work on optimizing the application for performance and usability. They can use testing frameworks like Jest or Mocha for this.
